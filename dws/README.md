# Day4 DWS 主题汇总层

## 本层位置

DWS 位于统一店铺月明细和业务指标之间。它只读取已经通过校验的 DWD，把同一份店铺事实整理为平台、区域和类目三个稳定主题。

DWD 回答“某个店铺当月发生了什么”，DWS 回答“按某个分析维度汇总后是什么结果”。这一层的存在可以避免 DM 和看板反复扫描千万级明细，也防止不同页面各自定义汇总口径。

## 步骤

### 1：确定三种主题模型

`sql/01_create_tables.sql` 定义平台月、区域月、类目月、类目恢复映射和类目 TOP10 五张结果表。

这一步先明确每种分析主题的粒度和唯一键，使同一指标在不同月份、平台和维度之间可比较。

### 2：形成平台月汇总

`dws_platform_month_summary_2026_gj` 按月份和平台汇总销售额、销量、店铺数以及有效数值行数，并保留上月销售额和环比。

平台月汇总是 DWS 的总量基准。DM 月度 KPI、平台分布和平台月度变化都以它为输入，区域和类目汇总也必须与它对平。

202601 在主分析期内没有上月，因此环比为 `NULL`；202602-202606 使用同平台前一个月计算。

### 3：形成区域月汇总

原始区域汇总先保留在 `dws_region_month_summary_2026_gj`，作为优化前基线。区域证据优化使用 `dim_region_standard_2026_gj` 固化标准行政区，再由 `dws_region_recovery_map_2026_gj` 保存行政层级、国家范围、店铺历史、完全地址和严格地址解析五类一对一证据。

最终结果写入 `dws_region_month_summary_v2_final_2026_gj`，在保留平台维度的同时生成全国、省级、市级和区县级四种区域结果。现有 DWD、原区域 DWS 和 DM 不被覆盖。

`region_level` 用来区分不同汇总层级：

| 层级 | 表示内容 | 主要用途 |
|---|---|---|
| 全国 | 平台当月全国总量 | 与平台月汇总对账 |
| 省级 | 省份小计 | 省份筛选和地图 |
| 市级 | 城市小计 | 城市筛选和下钻 |
| 区县级 | 具体区县 | 区县销售排行 |

这一设计防止全国、省、市、区县行被同时累加。解析顺序固定为原始有效地区、行政层级纠错、唯一店铺历史、完全地址、严格地址解析。美国等明确境外记录进入 `OVERSEAS`，港澳台进入 `SPECIAL_REGION`；没有足够证据的记录继续保留未知，不进行比例分配或模糊猜测。

地址解析至少需要两个相互一致的行政层级，且只接受唯一结果。每个恢复方法都保留证据行数和覆盖月份，最终区域总金额、销量和店铺数必须与 DWD 的 24 个分组完全一致。

区域证据链按以下成果连续形成：`06_create_region_recovery_tables.sql` 固化标准行政区和映射结构；`07_build_region_recovery_map.sql` 形成行政层级、国家范围、店铺历史和完全地址证据；`build_region_address_map.py` 补充严格地址证据；`load_region_v2_batches.py` 按24个事务批次形成区域 V2；`09_finalize_region_v2.sql` 生成看板使用的最终只读结果；`91_validate_region_v2.sql` 负责总量、覆盖率和境外隔离验收。

### 4：形成类目月汇总和 TOP10

`dws_category_month_summary_2026_gj` 保留每个月、每个平台的全部类目；`dws_category_month_top10_2026_gj` 在基础汇总之上生成平台内 TOP10 和销售额占比。

基础表负责保存完整事实并与平台总量对账，TOP10 只负责展示重点类目。两张表不能互相替代，因为 TOP10 本身不代表全部销售额。

类目汇总先使用 DWD 中的真实类目，再使用同平台唯一类目 ID 映射和同店铺唯一历史类目作为恢复证据。两类映射只从原始有效类目建立，不使用已经恢复的结果继续推断。仍没有可靠证据的记录归入“未分类”，不能按比例、均值或随机方式分摊。

`dws_category_recovery_map_2026_gj` 保存恢复依据、证据行数和覆盖月份，使每次回填可以追溯。Ozon 的真实类目来自源表 `category_name_new`；SMT、Amazon 和 Alibaba 只在映射满足一对一条件时回填。

基础类目汇总保留“未分类”以维持总金额完整并与平台汇总对账。TOP10 排名排除“未分类”，但销售额占比仍以包含未分类的全部销售额为分母，避免人为放大已分类类目的占比。

### 5：形成 2025 平台月同比基期

`dws_platform_month_summary_2025_gj` 从 2025 DWD 生成同口径的平台月汇总，共 6 个月、4 个平台、24 行。

2025 分支只承担平台月同比基期作用。它交给 DM 与 2026 平台月数据按同平台、同月份序号进行匹配，不用于当前区域和类目同比。

### 6：确认三种主题与 DWD 一致

DWS 验证：

- 平台月汇总覆盖 24 个 2026 分组；
- 平台月金额、销量和店铺数与 DWD 一致；
- 全国区域行与平台月汇总一致；
- 区域 V2 的 24 个分组与 DWD 金额、销量和店铺数差异为 0；
- 境外记录不进入中国区县排行，区域恢复映射不存在一对多结果；
- 包含“未分类”在内的全部类目金额与平台月汇总一致；
- 202601 环比保持 `NULL`；
- 类目 TOP10 排名在每个月、每个平台内唯一；
- 2025 平台月基期覆盖 24 个分组。

## 本层优化与修复记录

### 1. 类目恢复映射

类目优化没有把“未分类”按现有类目比例分摊，因为这种方式只能得到估算值，无法证明某个店铺真实属于哪个类目。

`dws_category_recovery_map_2026_gj` 只保存从原始有效类目得到的一对一证据：同平台类目 ID 只对应一个名称时形成 `CATEGORY_ID` 映射；同平台店铺六个月只出现一个有效类目时形成 `SHOP_HISTORY` 映射。解析顺序为真实类目、唯一类目 ID、唯一店铺历史、未分类，推断结果不会再次参与生成映射。

该过程使未分类占比由 23.1904% 降到 9.0111%，同时保持类目总金额、销量和店铺数与平台 DWS 对平。仍无证据的数据继续保留为“未分类”。

### 2. 区域恢复映射

`dim_region_standard_2026_gj` 提供标准省、市、区县及父子关系，`dws_region_recovery_map_2026_gj` 保存五类可追溯证据：

- `ADMIN_HIERARCHY`：修复省、市、区县字段错位；
- `COUNTRY_SCOPE`：区分中国大陆、境外、港澳台和范围未知；
- `SHOP_HISTORY`：同平台店铺六个月只有一个有效地区；
- `EXACT_ADDRESS`：完全相同地址只对应一个标准地区；
- `ADDRESS_PARSE`：地址中至少两个行政层级一致且结果唯一。

映射只使用原始有效地区生成，不使用已恢复结果继续推断。最终已定位区县销售额由 6140.05 亿元提高到 6283.28 亿元；Amazon US 的 8550.63 亿元全部归入境外范围，不再进入“三未知”或中国区县排行。

### 3. 区域批次断连和重复结果隔离

区域 V2 首次全量生成时客户端连接中断，但服务端查询稍后仍完成，随后重跑批次使过程表多出 156 条内容一致的全国层级记录。检查确认这些重复行的金额、销量和店铺数完全相同，并非业务事实重复。

项目没有删除或覆盖过程表，而是新建 `dws_region_month_summary_v2_final_2026_gj`，按声明的结果粒度折叠完全一致的行。最终表包含 79,236 行、24 个“月份 + 平台”分组，唯一粒度重复为 0，金额、销量和店铺数与 DWD 差异均为 0。

表的使用边界如下：

| 表 | 当前用途 | 是否作为看板数据源 |
|---|---|---|
| `dws_region_month_summary_2026_gj` | 优化前基线和前后对比 | 否 |
| `dws_region_month_summary_v2_2026_gj` | 批次构建过程和故障审计 | 否 |
| `dws_region_month_summary_v2_final_2026_gj` | 已去重、已验收的正式区域结果 | 是 |
| `dim_region_standard_2026_gj` | 标准行政区构建依据 | 否 |
| `dws_region_recovery_map_2026_gj` | 恢复证据和覆盖率审计 | 否 |

## 本层主要表

| 表 | 主题粒度 | 交给下游的内容 |
|---|---|---|
| `dws_platform_month_summary_2026_gj` | 月 + 平台 | 月度 KPI、环比和平台分布 |
| `dws_region_month_summary_2026_gj` | 月 + 平台 + 区域层级 | 区域优化前基线 |
| `dim_region_standard_2026_gj` | 标准行政区代码 | 区域父子关系和标准名称 |
| `dws_region_recovery_map_2026_gj` | 证据类型 + 平台 + 匹配键 | 可追溯的唯一地区恢复证据 |
| `dws_region_month_summary_v2_final_2026_gj` | 月 + 平台 + 地区范围 + 区域层级 | 看板地区筛选、排行、地图和覆盖质量 |
| `dws_category_month_summary_2026_gj` | 月 + 平台 + 类目 | 完整类目事实和多平台重汇总 |
| `dws_category_recovery_map_2026_gj` | 映射类型 + 平台 + 匹配键 | 类目 ID 和店铺历史的唯一证据映射 |
| `dws_category_month_top10_2026_gj` | 月 + 平台 + 类目排名 | 单平台类目 TOP10 |
| `dws_platform_month_summary_2025_gj` | 月 + 平台 | 2026 同比基期 |

## 与 DM 的关系

平台月汇总先提供稳定的总量和环比，DM 再把这些汇总转换为零售、物流和同比业务指标。区域和类目表直接服务看板下钻，但仍以平台月总量作为对账基准。
