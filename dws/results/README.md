# DWS 结果

## 目录作用

本目录保存平台、区域和类目汇总后的结果样本与对账证据。它要证明三种分析主题来自同一份 DWD，汇总维度虽然不同，但总金额、总销量和店铺口径能够相互解释。

## 结果

### 1. 平台月汇总

平台月结果展示 6 个月、4 个平台的金额、销量、店铺数、上月金额和环比。

它是 DWS 的总量基准，也是 DM 月度 KPI 的直接输入。预期共 24 个分组。

### 2. 区域月汇总

区域结果展示全国、省级、市级和区县级数据。保存 `region_level` 是为了证明各层级是不同分析口径，不能把多个层级直接相加。V2 另外保存 `region_scope` 和 `recovery_method`，用于区分国内、境外、港澳台、范围未知及证据来源。

区域全量文件较大时，验收只保留24组DWD对账、恢复方法覆盖率、优化前后区县定位金额、TOP15和代表性纠错样本，不重复导出完整结果。

### 3. 类目汇总与 TOP10

类目基础结果证明全部类目和“未分类”共同覆盖平台总销售额；TOP10 结果证明排名逻辑正确并支持看板展示。

TOP10 只能作为展示结果，完整性对账必须使用类目基础汇总。

### 4. DWS 对 DWD 

对账结果分别比较：

- 平台月 DWS 与 DWD；
- 全国区域汇总与平台月 DWS；
- 全部类目汇总与平台月 DWS。

这些说明 DWD 进入 DWS 时有没有丢失或重复累计。

### 5. 环比和排名检查

环比检查证明 202601 没有伪造上月基期，后续月份按同平台上月计算。排名检查证明同一月份、同一平台不存在重复类目名次。

### 6. 2025 平台月基期

2025 平台月结果证明同比分支已经从 DWD 进入 DWS。它必须覆盖 24 个分组并与 2025 DWD 对平，之后才能交给 DM 计算同比。

## 优化结果证据

### 类目优化

类目结果文件依次回答三个问题：优化前有多少“未分类”、恢复映射覆盖了哪些平台、优化后是否仍与平台总量一致。它们只保存汇总和映射统计，不导出完整 DWD。

### 区域优化

| 文件 | 说明 |
|---|---|
| `10_region_v2_integrity.csv` | 最终区域表分组、粒度和重复检查 |
| `11_region_v2_reconciliation.csv` | 24 个区域 V2 分组与 DWD 的金额、销量和店铺数对账 |
| `12_region_v2_scope_method.csv` | 国内、境外、港澳台、范围未知和恢复方法覆盖情况 |
| `13_region_v2_coverage_before_after.csv` | 修复前后已定位区县销售额对比 |
| `14_region_v2_map_summary.csv` | 五类区域恢复映射的数量和证据范围 |
| `14_region_v2_top15.csv` | 看板使用的中国大陆已定位区县 TOP15 样本 |
| `15_region_v2_correction_examples.csv` | 宝安、白云、浦东等层级纠错样本 |
| `16_region_v2_overseas_guard.csv` | Amazon US 是否完整进入境外范围 |
| `17_region_v2_validation_summary.csv` | 区域优化最终 PASS/FAIL 结论 |
| `region_address_parse_candidates.csv` | 严格地址解析候选及其唯一性依据 |

区域过程表曾因连接中断后服务端延迟完成而留下重复全国行，因此结果验收统一针对 `dws_region_month_summary_v2_final_2026_gj`，不针对过程表。结果文件保留这段修复证据，便于解释为什么存在 V2 和 V2 final 两张表。

## 结果

- 2026 平台月共 24 个分组；
- 2025 平台月共 24 个分组；
- 平台、全国区域和完整类目对账差异均为 0；
- 202601 环比为 `NULL`；
- TOP10 排名无重复。
