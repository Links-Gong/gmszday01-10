# 看板验收结果

## 目录作用

本目录保存 Day7-Day8 看板的可视化验收证据。数据库对账证明数字正确，本目录进一步证明这些数字在页面中能够被正确加载、筛选和阅读。

## 验收逻辑

### 1. 默认页面

桌面完整页面截图证明默认月份、全部平台和全部地区能够加载核心指标、图表、排行和地图。它代表用户第一次打开看板时的基本可用状态。

### 2. 单月筛选

月份筛选证据证明页面只有一个滑块点，并且滑块停在哪个月，全页就展示该月数据。指标变化应能与对应月份的 DM 或 DWS 结果核对。

### 3. 平台筛选

单平台或多平台截图证明平台条件能够传递到 KPI、平台分布、类目、企业和地区相关查询，而不是只改变筛选器文字。

### 4. 地区联动

省市联动截图证明省份会限制城市选项，城市会进一步限制区县排行和地图。同义行政区只显示一个标准选项，非法地区文本不进入列表。

### 5. 窄屏布局

窄屏截图证明指标、筛选器、图表标题和表格没有明显重叠或截断，页面仍可滚动和操作。

### 6. 数据状态

验收记录应覆盖销售额覆盖率提醒、同比无基期状态、无结果状态和数据库错误状态，证明页面不会用 0 或空白掩盖问题。

## 与 Day9 的连续关系

这些页面证据与 `validation/results/06_dashboard_checks.csv` 共同组成看板验收。截图证明视觉和交互，CSV 检查证明服务和组件状态；两者都通过后，Day9 才能给出全链路可交付结论。
